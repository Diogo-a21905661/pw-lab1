Relatório do Trabalho Laboratorial 1

O objetivo deste trabalho foi a criação de uma página web, seguido pela disponibilização dessa mesma página web num servidor na nuvem.

Introdução

Concluimos este objetivo através da resolução de 3 exercicios dados pelo enunciado:

  1. Alojamento da página Web na cloud.
  2. Conhecer a Internet.
  3. Acesso via HTTP à minha página Web.

Ao longo deste relatório, irei explicar a resolução destes exercicios, como também irei deixar links para os websites usados nas suas resoluções.


Exercicio 1

Começamos com a criação de uma página web base inicial, utilizando o código dado pelo professor no enunciado. A implementação desse código, no Visual Studio Code, levou à criação de uma página, que iremos utilizar para exercicios mais à frente.

Depois da criação da página web, tive que utilizar a plataforma online Heroku de modo a conseguir disponibilizar a pagina web previamente criada online. De modo a fazer isto, tive que criar ficheiros adicionais, ao lado do código HTML da página web, de modo a conseguir que o Heroku consiga disponibilizar a página.

Depois da implementação no Heroku, ficamos com esta Página Online.


Exercicio 2

(Endereços IP)

Neste exercicio vamos encontrar informações sobre os IPs do meu PC e do website da heroku, sendo essa informação os IPs em si e a sua localização.

De modo a encontrarmos o IP do computador, podemos, através do google, pesquisar "what is my ip", que dá o endereço público do computador. Depois, podemos localizar o IP através do website WhatIsMyIPAddress.



Desse mesmo modo, podemos encontrar o ip do website Heroku através do site Site24x7, que dá o IP do url que inserimos. Encontramos a localização da mesma forma que encontrámos no exercicio anterior.

Exercicio 3

(HTTP)

Neste exercicio aprendemos mais sobre como o HTTP, o protocolo de troca de mensagens entre um cliente e um servidor Web, funciona. Descobrimos assim que o protocolo HTTP funciona com duas mensagens, sendo essas o HTTP GET, onde o cliente pede uma determinada página Web ao servidor, e o HTTP POST, onde o cliente envia dados ao servidor. Esta é a arquitetura cliente-servidor.

Se quisermos, podemos utilizar o comando "CTRL + U" para visualizar o "código fonte" da página. Ao fazermos isto na Página Web criada no primeiro exercicio, vamos ficar com uma página auxiliar que contêm o código HTML que tinhamos copiado do enunciado no primeiro exercicio.



(Inspect)

Ao utilizar o comando e clicando na barra "Network", verificamos que precisamos de fazer um request à página de modo a termos a informação que queremos. De modo a fazer esse request, selecionei o URL do site e fiz "Enter".

No total, verificamos que aparecem três mensagens, ou seja, que são descarregados três ficheiros na sequencia de um clique num hiperlink. Nessas três mensagens, duas são ficheiros auxiliares que o cliente pediu ao servidor, sendo esses as imagens, e um é a determinada página Web em formato HTML, sendo esse o primeiro pedido HTTP GET.
